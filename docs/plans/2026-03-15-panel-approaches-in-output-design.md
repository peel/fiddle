# Panel: Include approaches in output format

## Problem

When the panel completes, its output (Consensus/Majority/No consensus/Recommendation) references approaches that the user was never shown. The approaches are only passed to the panel agents internally.

## Solution

Add a `### Approaches` section to the panel Output Format in `skills/panel/SKILL.md`, placed immediately after the `## Debate: <topic>` heading and before `### Consensus`. Each approach is rendered with its full text.

### Output format after change

```markdown
## Debate: <topic>

### Approaches

**Approach 1: <name>**
<full approach text as provided by caller>

**Approach 2: <name>**
<full approach text>

### Consensus
- ...

### Majority (2/3)
- ...

### No consensus
- ...

### Recommendation
...
```

### Behavior by invocation context

- **From orchestrate/brainstorming:** Approaches passed in from brainstorming — included verbatim.
- **Standalone:** Approaches generated by panel in "Before Phase 1" candidate generation — included as generated.

## Scope

Single file edit: `skills/panel/SKILL.md` Output Format section.
